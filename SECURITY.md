# Security Policy

## Reporting a vulnerability

If you believe you've found a security vulnerability in BibleBridge, please report it privately so we can fix it before it's disclosed publicly.

**Email:** security@holybible.dev

Please include:

- A description of the issue and its potential impact
- Steps to reproduce, a proof-of-concept, or the affected file and line(s)
- Your preferred disclosure timeline (if any)

**Response time:** Initial acknowledgment within 24 hours. Fix timeline depends on severity, typically within 7 days for high-severity issues.

## Scope

This policy covers:

- The standalone reader (this repository)
- The BibleBridge API backend at holybible.dev
- The reader running at holybible.dev/read

## Out of scope

- Social engineering of maintainers
- Physical attacks
- DoS / DDoS against public endpoints
- Third-party hosts running outdated copies of the standalone reader (please update to the latest release first)

## Safe harbor

Security research conducted in good faith — following this policy, without harming users or exfiltrating data beyond what's needed to demonstrate the issue — will not be pursued legally. We welcome your help.

Thank you for keeping BibleBridge safe.
